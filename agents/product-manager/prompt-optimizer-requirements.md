# 提示词迭代优化软件 - 需求分析

> 产出自：产品经理小李  
> 日期：2026-03-08

---

## 1. 目标用户 & 痛点

### 1.1 目标用户

| 用户群体 | 典型场景 |
|---------|---------|
| AI 应用开发者 | 需要产出稳定、可复现的 Prompt |
| 提示词工程师 | 职业优化 Prompt，追求效果最大化 |
| 企业 AI 负责人 | 批量优化内部 Prompt，确保质量 |
| AI 学习者 | 想学会怎么写好 Prompt |

### 1.2 核心痛点

| 痛点 | 描述 | 严重程度 |
|-----|------|---------|
| 盲目试错 | 写完 Prompt 全靠手调，效果靠玄学 | 🔴 高 |
| 反馈闭环慢 | 改了参数要反复手动测试，没有自动化 | 🔴 高 |
| 缺乏测试数据 | 不知道用什么 Case 验证 Prompt 质量 | 🟡 中 |
| 迭代靠人工 | 只能手动改，无法让模型自己优化自己 | 🔴 高 |
| 效果难量化 | 优化效果无法直观对比和改进 | 🟡 中 |

## 2. 核心功能 & 优先级

### 2.1 功能清单

| 优先级 | 功能模块 | 功能描述 |
|-------|---------|---------|
| **P0** | Prompt 优化引擎 | AI 分析当前 Prompt，给出改进建议或直接改写 |
| **P0** | 测试数据集管理 | 上传/维护测试用例（输入 + 期望输出） |
| **P0** | 自动迭代优化 | 循环：测试 → 评估 → 改写 → 测试，直到达标或达到迭代次数 |
| **P1** | 效果对比 | 新旧版本 Prompt 效果可视化对比 |
| **P1** | 版本管理 | Prompt 历史版本记录，支持回滚 |
| **P2** | 团队协作 | 团队共享测试集、Prompt 库 |
| **P2** | 模板市场 | 社区分享优质 Prompt 模板 |

### 2.2 MVP 定义（首版范围）

**只做 P0 功能，确保最小可用：**
1. 输入 Prompt + 测试数据 → 自动优化 → 输出优化后 Prompt + 测试报告
2. 支持手动单次优化 + 自动多轮迭代

## 3. 用户使用流程

### 3.1 主流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│ Step 1      │    │ Step 2      │    │ Step 3      │    │ Step 4      │
│ 输入初始    │ →  │ 准备测试    │ →  │ 启动优化    │ →  │ 验收交付    │
│ Prompt      │    │ 数据集      │    │ (自动迭代)  │    │             │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
                          ↑                                        │
                          └────────── 迭代循环 ◀───────────────────┘
```

### 3.2 详细步骤

| 步骤 | 用户动作 | 系统动作 |
|-----|---------|---------|
| 1 | 输入初始 Prompt | 解析 Prompt 结构 |
| 2 | 上传测试数据集（Excel/JSON）或手动输入 | 存储测试集，校验格式 |
| 3 | 设置优化目标（迭代次数/效果阈值） | 初始化优化任务 |
| 4 | 启动优化 | 循环执行：测试 → 评估 → 改写 → 测试 |
| 5 | 查看优化结果 | 输出最终 Prompt + 测试报告 + 效果对比 |
| 6 | 采纳 / 手动调整 / 继续迭代 | 版本记录 |

### 3.3 关键交互

- **测试数据集格式：**  
  ```json
  [
    {"input": "用户问题1", "expected": "期望回答1"},
    {"input": "用户问题2", "expected": "期望回答2"}
  ]
  ```
- **优化停止条件：**  
  - 达到最大迭代次数  
  - 效果提升达到阈值（如 90% 通过率）  
  - 效果不再提升

## 4. 竞品分析

### 4.1 竞品概览

| 产品 | 类型 | 核心特点 | 不足 |
|-----|------|---------|------|
| **PromptPerfect** | Web 服务 | 专业 Prompt 优化，支持多场景 | 收费贵，无自动化迭代 |
| **AIPRM** | Chrome 插件 | 社区共享 Prompts 库 | 偏向工具集合，无深度优化 |
| **PromptKnife** | 开源 | Prompt 版本管理、评审 | 无自动化优化能力 |
| **LangChain PromptHub** | 企业服务 | 团队协作、权限控制 | 偏管理，非优化 |

### 4.2 可借鉴设计

| 借鉴来源 | 借鉴点 |
|---------|-------|
| PromptPerfect | 场景分类（写作/代码/问答）、输出格式化 |
| AIPRM | 社区评分机制、模板市场 |
| PromptKnife | 版本管理、评审工作流 |
| LangChain | 团队权限管理、API 集成 |

### 4.3 差异化机会

1. **自动化迭代** — 竞品多为一键优化，无多轮迭代
2. **测试驱动** — 以测试数据为核心，而非纯生成
3. **效果量化** — 通过测试集通过率量化优化效果

---

## 5. 非功能需求

| 维度 | 要求 |
|-----|------|
| 响应时间 | 单次优化 < 30s（取决于模型） |
| 迭代速度 | 每轮迭代 < 1min |
| 数据安全 | 测试数据本地优先，不强制上传 |
| 可扩展性 | 支持自定义评估指标 |
